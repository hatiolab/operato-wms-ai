### 작업 내용

- 계획 수립 후 승인받아 구현 진행

### 변경/생성 파일
- `frontend/packages/operato-wes/client/pages/oms/shipment-order-detail.js` — 신규 생성 (약 750줄)
- `frontend/packages/operato-wes/client/route.ts` — shipment-order-detail import 추가

### 주요 결정 또는 발견 사항
- **컴포넌트 구조**: LitElement 팝업 (PageView 아님), `UiUtil.openPopupByElement()`로 호출
- **4탭 구성**: 기본정보 / 주문상세(Items) / 배송정보 / 할당내역
- **수평 타임라인**: 9단계 (등록→확정→할당→웨이브→인계→피킹→패킹→출하→마감), BACK_ORDER/CANCELLED 시 숨김
- **Lazy Loading**: 배송정보(Tab2)와 할당내역(Tab3)는 탭 최초 활성화 시 조회
- **상태별 액션**: REGISTERED(확정/취소), CONFIRMED(할당/취소), ALLOCATED(할당해제/취소), BACK_ORDER(재할당/취소), WAVED(취소), SHIPPED(마감)
- **11개 상태 배지**: 각 상태별 고유 색상 적용
- **부족수량 강조**: `shortQty > 0` 시 빨간색 표시
- 백엔드 트랜잭션 API (confirm/allocate/cancel 등)은 아직 미구현 — 프론트만 우선 구현
