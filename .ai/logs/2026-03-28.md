### 주요 결정 또는 발견 사항
- **컴포넌트 구조**: LitElement 팝업 (PageView 아님), `UiUtil.openPopupByElement()`로 호출
- **4탭 구성**: 기본정보 / 주문상세(Items) / 배송정보 / 할당내역
- **수평 타임라인**: 9단계 (등록→확정→할당→웨이브→인계→피킹→패킹→출하→마감), BACK_ORDER/CANCELLED 시 숨김
- **Lazy Loading**: 배송정보(Tab2)와 할당내역(Tab3)는 탭 최초 활성화 시 조회
- **상태별 액션**: REGISTERED(확정/취소), CONFIRMED(할당/취소), ALLOCATED(할당해제/취소), BACK_ORDER(재할당/취소), WAVED(취소), SHIPPED(마감)
- **11개 상태 배지**: 각 상태별 고유 색상 적용
- **부족수량 강조**: `shortQty > 0` 시 빨간색 표시
- 백엔드 트랜잭션 API (confirm/allocate/cancel 등)은 아직 미구현 — 프론트만 우선 구현

## 22:00 — OMS 테이블 불필요 필드 정리 (fulfillment 영역 분리)

### 작업 내용
- OMS 모듈의 테이블에서 fulfillment(출하 실행) 단계에서 결정되는 필드들을 제거
- OMS는 주문 수신/관리 단계이므로, 출하 시점에 확정되는 정보는 OMS 테이블에 불필요
- 설계 문서, 백엔드 엔티티, 프론트엔드 화면을 일괄 동기화

### 변경/생성 파일
- `docs/design/oms-table-design.md` — 3개 테이블에서 불필요 컬럼/인덱스 제거, ER 다이어그램·뷰 쿼리·이벤트 페이로드 정리
- `docs/design/oms-screen-design.md` — 와이어프레임에서 박스수/낱개수/송장번호/배송비/배송유형/택배사/서비스 제거
- `src/main/java/operato/wms/oms/entity/ShipmentOrderItem.java` — orderBoxQty, orderEaQty 필드 제거
- `src/main/java/operato/wms/oms/entity/ShipmentDelivery.java` — vehicleNo, invoiceNo, shippingFee, dlvType, carrierCd, carrierServiceType 필드 제거
- `src/main/java/operato/wms/oms/entity/ShipmentOrderStatus.java` — 뷰 SQL 및 trackingNo, carrierCd, carrierServiceType 필드 제거
- `src/main/java/operato/wms/oms/entity/ReplenishOrderItem.java` — orderBox, orderEa, resultBox, resultEa 필드 제거
- `frontend/packages/operato-wes/client/pages/oms/shipment-order-detail.js` — 박스수/낱개수 컬럼, 송장번호/배송유형/택배사 UI 제거

### 주요 결정 또는 발견 사항
- **shipment_order_items**: `order_box_qty`/`order_ea_qty`는 fulfillment 피킹 시점의 입수 환산 정보 → `order_qty` 하나로 충분

  - `vehicle_no`, `invoice_no`, `shipping_fee` → 출하 확정 시 결정되는 fulfillment 영역 정보
- **replenish_order_items**: `order_box`/`order_ea`/`result_box`/`result_ea` → `order_qty`/`result_qty`로 충분
