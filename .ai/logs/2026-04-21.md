## 10:30 — W1-I-2: 적치 추천 로케이션 API 구현

### 작업 내용
- StoragePolicy의 `putawayStrategy` 값(FIXED/ZONE/NEAREST/RANDOM)에 따라 빈 로케이션을 추천하는 서비스 메서드 구현
- REST API 엔드포인트 추가: `GET /rest/inbound_trx/putaway/recommend_locations`

### 변경/생성 파일
- `src/main/java/operato/wms/base/entity/StoragePolicy.java` — `PUTAWAY_STRATEGY_*` 상수 4개 추가
- `src/main/java/operato/wms/inbound/service/InboundTransactionService.java` — `recommendPutawayLocations()` 신규 메서드 추가
- `src/main/java/operato/wms/inbound/rest/InboundTransactionController.java` — `/putaway/recommend_locations` GET 엔드포인트 추가

### 주요 결정 또는 발견 사항
- FIXED: `Location.skuCd = :skuCd` 조건으로 고정 로케이션 조회
- ZONE: SKU의 `tempType` 기준으로 같은 온도 존 내 빈 로케이션 배정
- NEAREST: `sort_no ASC` 정렬로 도크에서 가장 가까운 빈 로케이션
- RANDOM: 화주사 전용 또는 공용 빈 로케이션 랜덤 배정
- occupied 로케이션 제외: 현재 재고가 있는 로케이션은 후보에서 제외

## 11:30 — W1-ST-2: 재고실사 완료 시 차이수량(diff_qty) 자동 계산 구현

### 작업 내용
- `/finish` 호출 시 `diff_qty = stocktake_qty - total_qty` 일괄 계산 후 업데이트
- 실사 헤더(`Stocktake`)의 `result_sku`(실사 완료 SKU 수)와 `diff_sku`(차이 발생 SKU 수) 집계

### 변경/생성 파일
- `src/main/java/operato/wms/stock/rest/StocktakeController.java` — `finishStocktask()` 메서드에 diff_qty 및 SKU 집계 로직 추가

### 주요 결정 또는 발견 사항
- `result_sku`: `stocktakeQty > 0`인 아이템의 distinct sku_cd 수
- `diff_sku`: `|diffQty| > 0.0001`인 아이템의 distinct sku_cd 수 (부동소수점 비교)
- Java Stream `filter().map(StocktakeItem::getSkuCd).distinct().count()` 패턴 활용
