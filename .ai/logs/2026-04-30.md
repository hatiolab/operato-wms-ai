## 10:40 — fulfillment-packing-pc.js 바코드 스캔 컴포넌트를 sku-barcode-input으로 교체

### 작업 내용
- 검수 화면의 상품 바코드 스캔 입력을 `ox-input-barcode` 직접 사용 → `sku-barcode-input` 컴포넌트로 교체
- `sku-barcode-input`이 API를 통해 바코드 → sku_cd를 해석하므로 상품 바코드(sku_barcd)도 인식 가능
- 바코드 직접 비교 방식 → API 해석 후 sku_cd 비교 방식으로 전환

### 변경/생성 파일
- `frontend/packages/operato-wes/client/pages/fulfillment/fulfillment-packing-pc.js` — `sku-barcode-input.js` import 추가, `_renderInspectionPanel()` 컴포넌트 교체, `_onBarcodeInput()` → `_onSkuSelect()` 핸들러 교체, 포커스 로직 수정

### 주요 결정 또는 발견 사항
- 기존 `_onBarcodeInput()` 에서 `item.barcode === barcode || item.sku_cd === barcode` 직접 비교 → 상품 바코드(sku_barcd)는 매칭 불가
- `sku-barcode-input`은 `sku-select` 이벤트로 `{ com_cd, sku_cd, sku_nm, barcode }` 반환 → `item.sku_cd === sku_cd` 매칭으로 전환
- `comCd`를 `selectedOrder?.com_cd`로 바인딩해 화주사 필터링 적용
- 포커스: `getElementById('barcodeInput')` → `querySelector('sku-barcode-input')?.focus()`
