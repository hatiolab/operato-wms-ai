## 6. 보안 설정 개선 — application-dev.properties

### 문제
- `spring.datasource.password`, `mail.smtp.password`가 평문으로 커밋됨

### 해결
1. **환경변수 치환 적용**
   ```properties
   spring.datasource.password=${DB_PASSWORD}
   mail.smtp.password=${MAIL_SMTP_PASSWORD}
   ```

2. **VSCode 디버그 환경변수 설정** (`.vscode/launch.json`)
   ```json
   "env": {
     "JAVA_HOME": "/Library/Java/JavaVirtualMachines/zulu-18.jdk/Contents/Home",
     "DB_PASSWORD": "hatioLAB1008",
     "MAIL_SMTP_PASSWORD": "1q2w3e4r~!"
   }
   ```

3. **Git 추적 중단**
   ```bash
   git rm --cached src/main/resources/application-dev.properties
   ```
   - `application-dev.properties`는 git index에서 제거 (로컬 파일은 유지)
   - `.vscode/`는 기존부터 untracked 상태 (안전)

### 수정 파일
- `src/main/resources/application-dev.properties`
- `.vscode/launch.json`

## 7. 보안 설정 개선 — application-prod.properties (Jasypt 암호화)

### 작업
- `application-prod.properties`의 DB 패스워드, 메일 패스워드를 Jasypt ENC() 형식으로 암호화

### Jasypt 설정 (application.properties)
```properties
jasypt.encryptor.password=${JASYPT_ENCRYPTOR_PASSWORD:operato-wms-secret}
```
- 마스터 키: 런타임 시 `JASYPT_ENCRYPTOR_PASSWORD` 환경변수로 주입
- 기본값 `operato-wms-secret`은 운영 환경에서 반드시 교체 필요

### 암호화 방법
- 알고리즘: `PBEWITHHMACSHA512ANDAES_256` (jasypt-spring-boot-starter 3.x 기본값)
- Jasypt 1.9.3 jar + bcprov-jdk15on-1.67 jar로 Java 프로그램 실행하여 암호화값 생성 및 검증

### 적용 결과
```properties
# 변경 전
spring.datasource.password=${DB_PASSWORD}
mail.smtp.password=${MAIL_SMTP_PASSWORD}

# 변경 후
spring.datasource.password=ENC(j0bkAPNVkbcvBTipXOqBSvfbh8SC6krCQqnzh1KHUkUgEFcq+z1LIffz/x0udRGc)
mail.smtp.password=ENC(gt0wCcfTVkgaqEz3aBWUASHpU9Og2u0CAX8EqQnVjVKIMUiP0rrW3brPJu+lLTS5)
```

### 운영 배포 시 주의사항
```bash
export JASYPT_ENCRYPTOR_PASSWORD=<강력한-운영-마스터키>
```
