## 6. 보안 설정 개선 — application-dev.properties

### 문제
- `spring.datasource.password`, `mail.smtp.password`가 평문으로 커밋됨

### 해결
1. **환경변수 치환 적용**
   ```properties
   spring.datasource.password=${DB_PASSWORD}
   mail.smtp.password=${MAIL_SMTP_PASSWORD}
   ```

2. **VSCode 디버그 환경변수 설정** (`.vscode/launch.json`)
   ```json
   "env": {
     "JAVA_HOME": "/Library/Java/JavaVirtualMachines/zulu-18.jdk/Contents/Home",
     "DB_PASSWORD": "hatioLAB1008",
     "MAIL_SMTP_PASSWORD": "1q2w3e4r~!"
   }
   ```

3. **Git 추적 중단**
   ```bash
   git rm --cached src/main/resources/application-dev.properties
   ```
   - `application-dev.properties`는 git index에서 제거 (로컬 파일은 유지)
   - `.vscode/`는 기존부터 untracked 상태 (안전)

### 수정 파일

```properties
jasypt.encryptor.password=${JASYPT_ENCRYPTOR_PASSWORD:operato-wms-secret}
```
- 마스터 키: 런타임 시 `JASYPT_ENCRYPTOR_PASSWORD` 환경변수로 주입

### 변경/생성 파일
- `docs/operations/frontend-docker.md` — 신규 생성
  - Dockerfile 구조 설명 (`hatiolab/operato-env:latest` 단일 스테이지)
  - 빌드 방법 3가지 (`yarn docker`, `docker build`, `buildx` 크로스 컴파일)
  - 실행 방법: 운영(5907), 로컬 테스트 `yarn docker:run`(4000:3000)
  - Docker Hub 푸시 (`yarn docker:push`)
  - 풀스택 연동 구성 (백엔드 9501 + 프론트엔드 5907)
  - 트러블슈팅: 의존성 오류, 메모리 부족, Apple Silicon 플랫폼 불일치

### 주요 결정 또는 발견 사항
- 베이스 이미지 `hatiolab/operato-env:latest`: Node.js + Yarn + Things Factory CLI 사전 설치된 커스텀 이미지
- `Dockerfile` 주석에 `EXPOSE 5907`이지만 `docker:run` 스크립트는 `4000:3000` 매핑 — 로컬 테스트용 편의 설정
- Things Factory `docker`, `docker:only` 스크립트: `things-factory-dockerize` CLI 래퍼
- Apple Silicon Mac에서 `--platform linux/amd64` 필요 (Things Factory 스크립트 기본 적용)
