## 23:30 — 웨이브 확정 시 피킹 지시 자동 생성 (이벤트 기반)

### 작업 내용
- 웨이브 확정 시 토털 피킹 데이터가 생성되지 않는 문제 해결
- 설계 문서에 명시된 이벤트 기반 아키텍처로 OMS-Fulfillment 간 연동 구현
- OMS가 웨이브 확정 시 WaveReleasedEvent 발행 → Fulfillment가 이벤트 구독하여 피킹 지시 자동 생성

### 변경/생성 파일
- `src/main/java/operato/wms/common/event/FulfillmentEvent.java` — 공통 이벤트 베이스 클래스 생성
- `src/main/java/operato/wms/common/event/WaveReleasedEvent.java` — 웨이브 릴리스 이벤트 클래스 생성
- `src/main/java/operato/wms/oms/service/OmsTransactionService.java` — releaseWave() 메서드에 이벤트 발행 추가
- `src/main/java/operato/wms/fulfillment/service/FulfillmentEventListener.java` — 이벤트 리스너 생성 (피킹 지시 자동 생성)

### 주요 결정 또는 발견 사항
- 설계 문서 확인: OMS(상위) → Fulfillment(하위)는 이벤트 발행으로 의존성 역전
- Spring ApplicationEventPublisher 사용하여 동일 JVM 내 이벤트 처리
- 이벤트 기반으로 느슨한 결합 유지 (OMS는 Fulfillment를 디펜던시로 가지지 않음)
- 웨이브 확정 시 pick_type(INDIVIDUAL/TOTAL)에 따라 피킹 지시 자동 생성
- 향후 WCS 연동 시에도 동일한 이벤트 구조 재사용 가능
