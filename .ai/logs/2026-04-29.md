# 2026-04-29 작업 로그

## 10:00 — sku-barcode-input.js 공통 컴포넌트 구현

### 작업 내용
- 바코드를 스캔하면 서버 API로 SKU 정보를 조회하는 공통 컴포넌트 신규 구현
- 결과 1건 → `sku-select` 커스텀 이벤트 즉시 dispatch
- 결과 복수 → 선택 팝업 표시 (모바일: 바텀 시트, PC: 중앙 모달)
- `ox-input-barcode`가 Shadow DOM으로 인해 `placeholder` 속성을 내부 input에 전달하지 않는 문제를 CSS 절대위치 오버레이 방식으로 해결
- `:focus-within` (Shadow DOM 내부 포커스도 감지)으로 포커스 시 placeholder 자동 숨김
- 스캔 완료 후 내부적으로 input 값 자동 초기화
- PDA 하드웨어 스캐너 연동을 위한 `focus()` 공개 메서드 추가

### 변경/생성 파일
- `frontend/packages/operato-wes/client/component/sku-barcode-input.js` — 신규 생성

### 주요 결정 또는 발견 사항
- `ox-input-barcode`는 카메라 스캔 내장 지원: `navigator.mediaDevices` 감지 시 바코드 아이콘 버튼 활성화, `@undecaf/zbar-wasm` (WebAssembly ZBar)으로 실시간 감지
- `ox-input-barcode`는 Shadow DOM 특성상 `placeholder` 속성을 내부 `<input>`에 전달하지 않음 → 절대위치 `.placeholder-hint` 오버레이로 대체
- 모바일/PC 통합 컴포넌트로 구현 (CSS `@media 768px`로 스타일 분기)
