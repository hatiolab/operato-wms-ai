# 2026-04-29 작업 로그

## 10:00 — sku-barcode-input.js 공통 컴포넌트 구현

### 작업 내용
- 바코드를 스캔하면 서버 API로 SKU 정보를 조회하는 공통 컴포넌트 신규 구현
- 결과 1건 → `sku-select` 커스텀 이벤트 즉시 dispatch
- 결과 복수 → 선택 팝업 표시 (모바일: 바텀 시트, PC: 중앙 모달)
- `ox-input-barcode`가 Shadow DOM으로 인해 `placeholder` 속성을 내부 input에 전달하지 않는 문제를 CSS 절대위치 오버레이 방식으로 해결
- `:focus-within` (Shadow DOM 내부 포커스도 감지)으로 포커스 시 placeholder 자동 숨김
- 스캔 완료 후 내부적으로 input 값 자동 초기화
- PDA 하드웨어 스캐너 연동을 위한 `focus()` 공개 메서드 추가

### 변경/생성 파일
- `frontend/packages/operato-wes/client/component/sku-barcode-input.js` — 신규 생성

### 주요 결정 또는 발견 사항
- `ox-input-barcode`는 카메라 스캔 내장 지원: `navigator.mediaDevices` 감지 시 바코드 아이콘 버튼 활성화, `@undecaf/zbar-wasm` (WebAssembly ZBar)으로 실시간 감지
- `ox-input-barcode`는 Shadow DOM 특성상 `placeholder` 속성을 내부 `<input>`에 전달하지 않음 → 절대위치 `.placeholder-hint` 오버레이로 대체
- 모바일/PC 통합 컴포넌트로 구현 (CSS `@media 768px`로 스타일 분기)

## 11:30 — resolveBarcode API에 skipInventory 옵션 추가

### 작업 내용
- `resolve_barcode` API에서 재고 바코드(inventories.barcode) 조회 단계를 선택적으로 건너뛸 수 있도록 옵션 추가
- 입고·VAS 등 재고가 아직 없는 화면에서 불필요한 재고 조회를 방지

### 변경/생성 파일
- `src/main/java/operato/wms/stock/service/StockTransactionService.java` — `resolveBarcode()` 에 `boolean skipInventory` 파라미터 추가
- `src/main/java/operato/wms/stock/rest/InvTransactionController.java` — 요청 바디 `skip_inventory` 필드 파싱 후 서비스에 전달
- `frontend/packages/operato-wes/client/component/sku-barcode-input.js` — `skipInventory` boolean property 추가, API body에 `skip_inventory` 포함
- `frontend/packages/operato-wes/client/pages/pda/pda-inbound-receiving.js` — `skipInventory` attribute 적용

### 주요 결정 또는 발견 사항
- `Boolean.parseBoolean(input.get("skip_inventory"))` — `Map<String, String>` 바디에서 문자열 `"true"` 파싱
- 화면별 기본값: 입고(`true`), 재고 이동/출고/피킹(`false` 기본값)
- API 파라미터명: `skip_inventory` (snake_case, Jackson SNAKE_CASE 컨벤션 준수)
