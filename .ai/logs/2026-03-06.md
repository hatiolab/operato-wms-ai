# 2026-03-06 작업 로그

## 1. Gradle 빌드 환경 구성

### 문제
- `operato-wms-ai` 프로젝트에 `gradlew` 및 `gradle-wrapper.jar` 없음
- 시스템 Gradle 9.3.1이 `otarepo-core` 빌드 스크립트와 호환되지 않음
- 현재 JVM: Java 8 (기본), Java 11, Java 18 설치됨 (Java 17 없음)

### 해결
- 시스템 Gradle로 Gradle 8.5 wrapper 생성
  ```bash
  JAVA_HOME=$(/usr/libexec/java_home -v 18) gradle wrapper --gradle-version 8.5
  ```
- 생성된 파일: `gradlew`, `gradlew.bat`, `gradle/wrapper/gradle-wrapper.properties`, `gradle/wrapper/gradle-wrapper.jar`

### 빌드 명령
```bash
JAVA_HOME=$(/usr/libexec/java_home -v 18) ./gradlew build
```

### 결과
- `BUILD SUCCESSFUL in 14s`
- `:otarepo-core:compileJava`, `:compileJava`, `:bootJar` 등 9개 태스크 성공

## 2. VSCode 디버그 환경 구성

### 생성 파일

#### `.vscode/launch.json`
- **operato-wms-ai (debug)**: `dev` 프로파일로 직접 실행
  - mainClass: `operato.wms.OperatoWmsApplication`
  - args: `--spring.profiles.active=dev`
  - vmArgs: `-Xmx512m`
  - JAVA_HOME: `/Library/Java/JavaVirtualMachines/zulu-18.jdk/Contents/Home`
- **operato-wms-ai (attach)**: 원격 디버거 연결 (localhost:5004)

#### `.vscode/settings.json`
- `java.jdt.ls.java.home`: zulu-18 JDK 경로 지정
- `java.configuration.runtimes`: JavaSE-18 기본 런타임 등록
- `java.configuration.updateBuildConfiguration`: automatic

### 디버그 실행 시 "Build failed" 이슈
- 원인: VSCode Java 확장이 사용할 JDK 경로 미설정
- 해결: `.vscode/settings.json`에 JDK 경로 추가
- 조치: `Java: Clean Java Language Server Workspace` 후 VSCode 재시작
