### 변경/생성 파일
- `src/main/java/xyz/elidom/base/system/meta/MenuMetaService.java` (otarepo-core) — REF_TYPE_URL 분기 구현

### 주요 결정 또는 발견 사항
- `HttpUtil.executeGetMethod()` → 인증 없는 Raw HTTP
- `HttpSender.sendByGet(url, null, options)` → `Authorization-Type: json`, `Authorization-Key: Base64(userJson)` 헤더로 인증
- `RestAuthenticationFilter`의 `doJsonAuth()`가 Base64 인코딩된 User JSON을 인증 처리
- refName이 `/`로 시작하는 상대 경로인 경우 현재 요청의 `scheme://host:port`를 자동으로 앞에 붙임
- 현재 컨텍스트(`domain_id`, `locale`)를 쿼리 파라미터로 자동 추가
- 도메인 파악은 `x-things-factory-domain` 헤더 → `Referer` 경로 순으로 동작함을 확인
- `HttpSender`가 `AUTH_TYPE`/`AUTH_KEY` 외 헤더 미지원 — Referer 추가는 HttpSender 수정 필요
