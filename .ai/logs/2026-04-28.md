## 10:00 — W23-IR-1, W23-IR-2: 검수 반려 API 및 반려 재고 처리 구현

### 작업 내용
- 입고 검수 중 반려가 발생했을 때 REJECTED 상태로 전환하고 불량 로케이션에 재고를 생성하는 기능 구현
- 자동 마감 체크 SQL에서 REJECTED 상태도 완료로 간주하도록 수정

### 변경/생성 파일
- `src/main/java/operato/wms/inbound/WmsInboundConstants.java` — `STATUS_REJECTED = "REJECTED"` 상수 추가
- `src/main/java/operato/wms/inbound/service/InboundTransactionService.java` — `rejectReceivingOrderLine()`, `processRejectedReceivingItem()` 신규 메서드 추가, `closeReceivingOrder()`에서 REJECTED 아이템 skip 처리
- `src/main/java/operato/wms/inbound/rest/InboundTransactionController.java` — hook 상수 2개 추가, `POST receiving_orders/line/{id}/reject` + `POST receiving_orders/line/reject` 엔드포인트 추가, 자동 마감 체크 SQL에 `STATUS_REJECTED` 조건 추가

### 주요 결정 또는 발견 사항
- `processRejectedReceivingItem()`: DEFECT 로케이션(loc_type = 'DEFECT')이 없으면 불량 재고 생성 없이 반환 (선택적 처리)
- `createInventoriesByReceivingOrder()`는 이미 STATUS_END만 처리하므로 REJECTED는 자동 제외됨
- 반려 수량은 rcvQty 기준으로 불량 재고 생성 (검수 중 확인된 수량)

- 트리거링은 프레임워크가 담당하므로 Job 로직과 설정 상수만 구현

### 변경/생성 파일
- `src/main/java/operato/wms/oms/WmsOmsConfigConstants.java` — 자동 웨이브 설정 상수 6개 추가 (enabled, group_by, pick_type, max_order_count, wh_cd, com_cd)

- jobs 테이블에 `handler_type=static`, `handler=operato.wms.oms.job.OmsWaveJob` 등록으로 활성화
- `@EnableScheduling`은 `OperatoWmsApplication.java`에 이미 선언됨
