## 22:30 — 업로드 MalformedStreamException 수정 (Spring multipart 설정)

### 작업 내용
- `Stream ended unexpectedly` 에러 수정 시도
- Spring Boot 3.x에서 `spring.http.multipart.*`가 무시됨을 발견 → `spring.servlet.multipart.*`로 수정

### 변경/생성 파일
- `otarepo-core/src/main/resources/application.properties` — `spring.servlet.multipart.max-file-size=500MB`, `spring.servlet.multipart.max-request-size=500MB` 추가

### 주요 결정 또는 발견 사항
- Spring Boot 2.x: `spring.http.multipart.*` / Spring Boot 3.x: `spring.servlet.multipart.*` — 기존 설정이 무시되어 Tomcat 기본값(1MB) 적용 중이었음
- 단, 실제 원인은 Koa 프록시 레이어였음 (다음 섹션)

## 23:30 — 다운로드 파일 손상 문제 수정 (Koa 프록시 바이너리 응답)

### 작업 내용
- 다운로드한 파일이 깨져서 열리지 않는 문제 수정
- 원인: `processRest`에서 axios 기본 responseType(`json`)으로 받아 바이너리를 UTF-8 텍스트 디코딩 → 손상
- `responseType: 'arraybuffer'`로 변경, JSON 응답만 파싱, 나머지는 Buffer 그대로 반환

### 변경/생성 파일
- `frontend/packages/operatofill/server/middlewares/index.ts` — `processRest()` 수정: `responseType: 'arraybuffer'`, Content-Type/Content-Disposition 헤더 전달, JSON/바이너리 분기 처리
- `frontend/packages/operatofill/dist-server/middlewares/index.js` — TypeScript 빌드 결과 반영

### 주요 결정 또는 발견 사항
- axios `responseType: 'arraybuffer'` → `Buffer.from(response.data)` 로 바이너리 손상 없이 전달
- `content-type`, `content-disposition`, `content-length` 헤더를 백엔드에서 클라이언트로 그대로 전달하여 파일명·타입 정확성 보장
