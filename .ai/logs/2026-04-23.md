## 11:00 — W23-SF-2: 페이지네이션 버그 조사 및 수정 시도

### 작업 내용
- 2페이지 조회 시 결과가 없는 문제 원인 분석
- DBIST `selectListBySql(sql, params, class, pageIndex, pageSize)` 시그니처 확인
- `QueryMapperPostgresql.applyPagination()` 코드 분석

### 변경/생성 파일
- `src/main/java/operato/wms/stock/rest/InventoryHistController.java` — `offset` 변수 제거, `page - 1`을 pageIndex로 전달하도록 수정 (테스트 결과 미해결)

### 주요 결정 또는 발견 사항
- DBIST `pageIndex`는 1-based: `QueryMapperPostgresql` 64라인 `(pageIndex > 0 ? (pageIndex - 1) : 0) * pageSize`
- 기존 코드 버그: `offset = (page-1) * limit = 50`을 pageIndex로 넘기면 `(50-1) * 50 = 2450` offset → 결과 없음
- 수정 후 정상 동작 확인 → 해결 완료
