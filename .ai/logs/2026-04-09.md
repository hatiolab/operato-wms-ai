## 14:00 — 출하 확정·마감 시 재고 차감 및 shipped_qty 반영

### 작업 내용
- OMS 출하 주문 할당 시 `inv_qty`(재고 수량)와 `reserved_qty`(할당 수량)를 고려한 FEFO 할당 로직 확인
- 할당 후 `reserved_qty` 업데이트가 이미 구현되어 있음을 확인 (별도 수정 불필요)
- 출하 확정 시 `inv_qty` 차감이 누락된 문제 발견 → `closeShipmentOrder` (SHIPPED→CLOSED) 시점에 재고 차감 로직 추가
- 출하 확정 시 `ShipmentOrderItem.shipped_qty` 미반영 문제 발견 → `confirmShipping`에서 `stock_allocations` 집계 후 업데이트 추가
- 출하 취소(`cancelShipping`) 로직 수정: `inv_qty`는 마감 시 차감되므로 취소 시엔 `reserved_qty`만 해제, `shipped_qty` 0 초기화

### 변경/생성 파일
- `src/main/java/operato/wms/fulfillment/service/FulfillmentShippingService.java` — `confirmShipping`에 `shipped_qty` 업데이트, `cancelShipping`에 `reserved_qty`만 해제 + `shipped_qty` 0 초기화
- `src/main/java/operato/wms/oms/service/OmsShipmentOrderService.java` — `closeShipmentOrder`에 `inv_qty` 차감 + `reserved_qty` 해제 + `stock_allocations` 상태 RELEASED 변경 로직 추가

### 주요 결정 또는 발견 사항
- **재고 생명주기 확정**: `reserved_qty` 증가(할당) → `shipped_qty` 반영(출하 확정) → `inv_qty` 차감 + `reserved_qty` 해제 (주문 마감)
- `confirmShipping` 시점에는 `inv_qty` 차감하지 않음. 마감 이전에 주문이 취소될 수 있기 때문
- `selectListBySql` 반환 타입이 raw `List<Map>`이므로 제네릭 파라미터 없이 사용해야 컴파일 오류 없음
