## 1. 시스템 개요

Operato WMS는 자가 물류 및 3PL(Third Party Logistics) 운영을 위한 클라우드 기반 창고 관리 시스템이다.

### 핵심 특징

| 특징 | 설명 |
|------|------|
| 클라우드 / Web 기반 | 브라우저 접근, PWA 모바일 환경 지원 |
| 멀티 테넌시 | 물리적 창고를 논리적 사이트(도메인)로 구분, 화주사별 별도 환경 |
| 3PL 지원 | 화주사, 창고, 협력사별 옵션 처리 |
| 실시간 현황 파악 | 입출고, 재고, 피킹 현황 실시간 조회 |
| 선입선출 지원 | 출고 시 시스템에서 FIFO 자동 처리 |
| 외부 설비 연계 | DAS, DPS 등 표시기 기반 설비, 택배사 API 연동 |
| 출력물 자동 생성 | 입고지시서, 거래명세서, 바코드 라벨, 피킹지시서 등 |

## 2. 사용자 및 역할

| 역할 | 설명 | 주요 사용 장비 |
|------|------|--------------|
| Manager (운영자) | 시스템 설정, 입출고 지시, 현황 조회, 정산 관리 | PC 브라우저 |
| Operator (작업자) | 피킹, 적치, 재고 이동, 검수 처리 | PDA / 태블릿 |
| 화주사 담당자 | 입출고 계획 등록, 실적 조회, 재고 현황 확인 | PC 브라우저 |

## 4. 업무 프로세스

### 4.1 B2C 출고 프로세스

**특징**: 개인 소비자 대상 소량 다품종 출고. 합포장, 복수 주문 처리. DPS 연계 가능.

```
[SO 수신/접수 (API or Excel)]
    ↓
[주문 유형 분석]
    ├─ Single (낱개 1EA, 동일 패턴)
    │       ↓
    │   [보충 지시 (S)] → [보충 지시서 출력] → [보충 확정]
    │       ↓
    │   [피킹 지시 생성] → [피킹지시서 출력] → [택배 송장 출력]
    │       ↓
    │   [매뉴얼 피킹]
    │       ↓
    │   [피킹 확정]
    │
    └─ Complex (합포장, 복수 주문)
            ↓
        [보충 지시 (C)] → [보충 지시서 출력] → [보충 확정]
            ↓
        [피킹 지시 생성] → [피킹지시서 출력]
            ↓
        [DPS 전송] → [DPS 피킹] → [피킹 확정]
            ↓
        [스캔 검수]
            ↓
[출고 확정] → [출고 실적 완료]
```

**작업 횟수**: 총 3회 (보충, 피킹, 스캔 검수)

## 5. 비기능 요구사항

| 분류 | 요구사항 |
|------|---------|
| **플랫폼** | Web 기반 (브라우저 접근), PWA 지원으로 모바일(PDA, 태블릿) 환경 대응 |
| **멀티 테넌시** | 화주사/창고별 독립적인 도메인(사이트) 환경 제공 |
| **확장성** | 화주사별 옵션 처리, 커스텀 서비스 훅(pre/post hook)을 통한 업무 확장 |
| **외부 연동** | OMS/ERP I/F, 택배사 API (CJ, 롯데, 우체국 등), DAS/DPS 설비 연동 |
| **보안** | 역할 기반 접근 제어 (RBAC), 메뉴별 권한 관리 |
| **운영** | 로그 관리, 공통 코드 관리, 다국어 지원 (메시지 관리) |
| **출력** | 바코드 라벨, 입고지시서, 피킹지시서, 거래명세서, 인수증 등 Report 출력 |
| **클라우드** | 클라우드 서비스 기반으로 시스템 유지보수 효율 증대 |
