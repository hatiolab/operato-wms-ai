## 1. 배경: AI 시대의 WMS 패러다임 전환

### 1.1 글로벌 시장 동향

물류·WMS 분야에서 AI 도입은 2025년을 분기점으로 급속히 확산되고 있다.

| 지표 | 수치 |
|------|------|
| 물류 AI 시장 규모 (2028년 전망) | **$83.3B** (연평균 성장률 47.5%) |
| WMS에서 AI를 표준 기능으로 채택한 비율 (2026) | 증가 추세 — "AI는 이제 선택이 아닌 필수" |
| 주요 WMS 벤더 대응 | Oracle, SAP, Infor, Made4net 모두 AI 로드맵 발표 완료 |

### 1.2 패러다임 변화: 3단계 진화

```
1세대 WMS (기록)          2세대 WMS (최적화)         3세대 WMS (자율·지능)
─────────────────         ─────────────────          ─────────────────────
입출고 전산화              규칙 기반 자동화             AI 기반 예측·의사결정
재고 현황 파악             피킹 경로 최적화             자율 에이전트 운영
바코드 기반 추적           WCS/설비 연동               자연어 인터페이스
수작업 → 시스템화          반자동화                    Physical AI·로봇 통합
```

**현재 Operato WMS는 1.5세대 ~ 2세대 수준**이며, 3세대로의 전환을 위한 로드맵이 필요하다.

### 1.3 핵심 트렌드 요약

| 트렌드 | 설명 |
|--------|------|
| **AI 의사결정 계층** | AI가 WMS의 모든 판단(적치, 피킹, 보충)에 개입하는 표준 레이어로 자리잡음 |
| **수요 예측** | 계절성, 이벤트, 외부 데이터를 반영한 ML 기반 입출고 예측 |
| **슬로팅 최적화** | 상품 인기도·이동 패턴 기반 로케이션 자동 재배치 |
| **생성형 AI (GenAI)** | 자연어로 재고 조회, 이상 감지, 보고서 자동 생성 |
| **자율 에이전트 (Agentic AI)** | 인간 개입 없이 보충 지시·적치 결정·이상 처리를 스스로 수행 |
| **디지털 트윈** | 창고 레이아웃·동선의 3D 시뮬레이션으로 사전 최적화 |
| **컴퓨터 비전** | 카메라 기반 재고 계수, 파손 감지, 입고 자동 검수 |
| **AMR/로봇 통합** | AI 경로 계획 기반 자율 이동 로봇과 WMS 연동 |
| **지속가능성** | 탄소 발자국 추적, 친환경 물류 경로 최적화 |

## 2. 현재 Operato WMS 수준 평가

### 2.1 현재 강점

| 영역 | 현황 |
|------|------|
| 기본 WMS 기능 | 입고·출고·재고·실사 전 영역 커버 |
| 멀티 테넌시 | 화주사별 독립 도메인 지원 (3PL 적합) |
| 외부 연동 | 택배사 API, DAS/DPS 설비 연동 기반 보유 |
| 커스텀 훅 | pre/post 커스텀 서비스로 확장 가능한 구조 |
| 출력물 | 입고지시서, 거래명세서, 피킹지시서 등 |

### 2.2 현재 한계 (AI 관점)

| 영역 | 현재 | AI 시대 요구사항 |
|------|------|----------------|
| 의사결정 | 사람이 모든 결정 (수동 피킹 지시, 수동 적치) | AI 추천·자동화 |
| 데이터 활용 | 현황 조회만 가능 (과거 데이터 미활용) | 예측·최적화에 활용 |
| 인터페이스 | 화면 기반 CRUD | 자연어 질의, 챗봇 |
| 이상 감지 | 없음 (수동 발견) | 실시간 자동 감지·알림 |
| 재고 예측 | 없음 | ML 기반 수요 예측 |
| 슬로팅 | 수동 로케이션 관리 | AI 기반 자동 최적화 |
| 로봇 연동 | DAS/DPS 표시기 연동 수준 | AMR 완전 통합 |

## 3. 추가해야 할 기능 — 트렌드 분석

### 3.1 🤖 AI 예측 및 최적화

#### 3.1.1 수요 예측 (Demand Forecasting)

**현황**: 없음
**도입 배경**: AI는 과거 입출고 이력, 계절성, 외부 이벤트를 분석해 재고 수요를 예측. Oracle, Infor, Made4net 모두 핵심 AI 기능으로 출시.

**기능 내용**:
- 상품별 출고 패턴 ML 학습 (과거 12~24개월 데이터)
- 계절성/이벤트 반영 출고 예측 (다음 30/60/90일)
- 재고 부족 사전 경고 (예상 재고 소진일 표시)
- 과잉 재고 감지 및 반품·이전 추천

**데이터 소스**: `inventory_hists` (이력), `release_order_items` (출고 패턴), 외부 캘린더 API

---

#### 3.1.2 AI 슬로팅 최적화 (Intelligent Slotting)

**현황**: 수동 로케이션 배치
**도입 배경**: AI 기반 슬로팅으로 피커 이동 거리 20~35% 단축 (업계 사례). 출고 빈도·상품 크기·함께 자주 주문되는 상품(동반 주문)을 기반으로 자동 재배치.

**기능 내용**:
- 상품 ABC 분석 (출고 빈도 기반 A/B/C 등급 자동 산출)
- AI 기반 최적 로케이션 추천 (피킹존 근접 배치)
- 동반 주문 분석 — 함께 출고되는 상품 인근 배치
- 계절별 슬로팅 자동 재조정 스케줄링
- 재배치 비용 vs 효율 향상 ROI 시뮬레이션

**데이터 소스**: `release_order_items`, `picking_order_items`, `inventories`, `locations`

---

#### 3.1.3 AI 피킹 경로 최적화 (Pick Path Optimization)

**현황**: 수동 피킹 순서 / 기본 Wave 단위 지시
**도입 배경**: AI가 창고 레이아웃, 현재 재고 위치, 작업자 위치를 고려해 최단 피킹 경로를 실시간 계산.

**기능 내용**:
- 피킹 지시 시 최단 경로 자동 계산 (TSP 기반)
- 멀티 오더 배치 피킹 최적화 (Batch Picking)
- 혼잡 구간 회피 (동시 작업자 위치 기반 경로 조정)
- 피킹 완료 예상 시간 예측

---

#### 3.1.4 노무 인력 예측 (Labor Forecasting)

**현황**: 없음
**도입 배경**: Infor, Made4net이 핵심 AI 기능으로 출시. 입출고 예정 데이터 기반 필요 인원 자동 계산.

**기능 내용**:
- 출고 주문량 기반 필요 작업 인원 예측 (당일/익일/주간)
- 작업 유형별 (피킹, 포장, 검수) 인력 배분 최적화
- 초과 근무 사전 경고

---

### 3.2 💬 생성형 AI & 자연어 인터페이스

#### 3.2.1 AI 챗봇 / 자연어 질의 (WMS Copilot)

**현황**: 없음
**도입 배경**: Logistics Reply의 LEApedia, Oracle WMS AI Assistant 등 대형 WMS 벤더들이 LLM 기반 자연어 인터페이스를 표준 탑재. 관리자가 화면을 탐색하지 않고 자연어로 즉시 데이터 조회 가능.

**기능 내용**:
- 자연어로 재고·입출고 현황 조회
  - "어제 A화주사 출고 총량이 얼마야?"
  - "P001 상품이 어느 로케이션에 있어?"
  - "이번 주 피킹 완료율 보여줘"
- 이상 상황 요약 보고 자동 생성
- 작업 지시 자연어 입력 ("A화주사 내일 입고 지시 만들어줘")
- RAG(Retrieval Augmented Generation) 기반 WMS 내부 데이터 연결

**기술**: Claude API (Anthropic) + RAG, Spring Boot REST API

---

#### 3.2.2 AI 보고서 자동 생성

**현황**: 수동 Excel 다운로드 수준
**기능 내용**:
- 일간/주간/월간 운영 리포트 자동 생성 (자연어 요약 포함)
- 화주사별 입출고 실적 요약 자동 이메일 발송
- 이상 지표 자동 감지 및 하이라이트 (KPI 대비 이탈)

---

### 3.3 🔍 이상 감지 및 예측 유지보수

#### 3.3.1 실시간 이상 감지 (Anomaly Detection)

**현황**: 없음 (수동 발견)
**도입 배경**: Infor가 2025년 ML 기반 이상 감지를 WMS에 내장. 피킹 오류, 재고 불일치, 처리 속도 이상을 실시간 감지.

**기능 내용**:
- 재고 수량 급변 감지 (정상 범위 벗어난 입출고)
- 피킹 오류율 이상 감지 (작업자/구역별 오류 패턴)
- 적치 후 오랫동안 이동 없는 재고 감지 (Dead Stock 경고)
- 유통기한 임박 재고 자동 알림 (현재 임박 여부 필드는 있으나 알림 미구현)

---

#### 3.3.2 재고 실사 AI 보조 (AI-Assisted Stocktake)

**현황**: 수동 실사 (StocktakeController에 핵심 로직 미구현)
**기능 내용**:
- AI가 실사 우선순위 상품 자동 추천 (이상 감지된 상품, 고가 상품)
- 카메라 기반 자동 재고 계수 (Computer Vision 연동)
- 실사 결과 이상 자동 분석 (차이가 큰 항목 원인 추론)

---

### 3.4 🏭 로봇·설비 통합

#### 3.4.1 AMR(자율 이동 로봇) 연동

**현황**: DAS/DPS 표시기 연동 수준
**도입 배경**: 2026년부터 AMR + WMS 완전 통합이 업계 표준. AI 경로 계획 기반 AMR이 피킹·보충 작업을 자율 수행.

**기능 내용**:
- AMR 작업 지시 API 표준 인터페이스 (ROS 2 / Fleet Management API)
- AMR 위치 실시간 모니터링 및 WMS 연동
- AI 기반 AMR 태스크 우선순위 자동 배정
- 인간-로봇 협업 구역 동적 관리

---

#### 3.4.2 컴퓨터 비전 (Computer Vision)

**현황**: 없음
**기능 내용**:
- 입고 시 카메라 기반 자동 검수 (수량 계수, 파손 감지)
- 바코드/QR 자동 인식 (스캔 없이 카메라로 처리)
- 재고 실사 자동화 (드론·카메라 기반 재고 계수)

---

### 3.5 📊 고급 분석 및 가시성

#### 3.5.1 AI 대시보드 & 실시간 KPI

**현황**: 기본 현황 조회 (정적 화면)
**기능 내용**:
- 실시간 운영 KPI 대시보드 (입고율, 출고 정확도, 피킹 속도)
- 예측 KPI (오늘 출고 완료 예상 시각, 병목 구간 예측)
- 화주사별 SLA 달성률 실시간 모니터링
- 드릴다운 분석 (구역별 → 로케이션별 → 작업자별)

---

#### 3.5.2 디지털 트윈 (Digital Twin)

**현황**: 기본 3D 재고 모니터링 언급 (ref 문서)
**도입 배경**: AI 기반 디지털 트윈으로 창고 레이아웃 변경 전 시뮬레이션 가능. 슬로팅 변경, 구역 재배치 효과를 사전 검증.

**기능 내용**:
- 창고 레이아웃 3D 가상 모델
- 슬로팅 변경 시뮬레이션 (변경 전 예상 효율 계산)
- 작업자 동선 히트맵 (혼잡 구간 시각화)
- "What-if" 시나리오 분석 (로케이션 추가, 구역 변경 효과)

---

### 3.6 🌱 지속가능성 (Sustainability)

**현황**: 없음

**기능 내용**:
- 입출고 트랜잭션별 탄소 배출량 추적
- 친환경 피킹 경로 최적화 (이동 거리 최소화)
- 화주사별 탄소 발자국 리포트 자동 생성
- 포장재 최적화 추천 (적정 박스 사이즈 AI 추천)

---

### 3.7 🔗 연동 및 플랫폼 확장

#### 3.7.1 OMS/ERP AI 연동

**기능 내용**:
- 실시간 양방향 API (현재 단방향 수신 수준)
- AI 기반 주문 이상 자동 감지 (중복 주문, 비정상 주문 패턴)
- 이커머스 플랫폼 직접 연동 (Shopify, Coupang, Naver 등)

#### 3.7.2 실시간 배송 추적 통합

**기능 내용**:
- 택배사 배송 상태 실시간 조회 (현재 송장 번호 관리 수준)
- 배송 이상 감지 및 자동 알림 (배달 지연, 반송)
- 고객사 자가 배송 추적 포털 (화주사 셀프서비스)

| # | 기능 | 세부 내용 | 우선순위 |

## 5. 기술 아키텍처 방향

### 5.1 AI 레이어 추가 방향

현재 Spring Boot 모놀리식 구조에 AI 기능을 추가할 때의 권장 아키텍처:

```
┌──────────────────────────────────────────────────────────┐
│                     Client Layer                          │
│         Web UI  /  PDA App  /  WMS Copilot Chat          │
└──────────────────────────┬───────────────────────────────┘
                           │
┌──────────────────────────▼───────────────────────────────┐
│                  Operato WMS API (Spring Boot)            │
│    기존 REST API  +  AI Gateway API  +  Event Publisher  │
└────────┬───────────────────────────────────┬─────────────┘
         │                                   │
┌────────▼────────┐              ┌───────────▼─────────────┐
│  AI Service     │              │   Event Stream (Kafka)   │
│  (Python/FastAPI│              │   실시간 이상 감지         │
│  or Java)       │              │   AI 파이프라인 트리거     │
│                 │              └─────────────────────────┘
│ ┌─────────────┐ │
│ │ LLM (Claude)│ │              ┌─────────────────────────┐
│ │ + RAG       │ │              │   Data Warehouse         │
│ └─────────────┘ │              │   (입출고 이력, ML 학습용)│
│ ┌─────────────┐ │              └─────────────────────────┘
│ │ ML Models   │ │
│ │ (수요예측,   │ │
│ │  슬로팅)    │ │
│ └─────────────┘ │
└─────────────────┘
```

### 5.2 WMS Copilot 구현 방향 (Phase 2.1 세부)

```
사용자 자연어 입력
    ↓
Claude API (claude-opus-4-6 or claude-sonnet-4-6)
    ↓
RAG: WMS 내부 데이터 검색
├── inventory_hists (재고 이력)
├── release_orders (출고 현황)
├── receivings (입고 현황)
└── stocktakes (실사 결과)
    ↓
응답 생성 (자연어 + 데이터 테이블)
```

**기술 스택**:
- LLM: Claude API (`claude-sonnet-4-6`)
- Embedding: Anthropic Embeddings 또는 OpenAI Ada
- Vector DB: pgvector (PostgreSQL 확장) 또는 Pinecone
- Framework: Spring AI 또는 Python FastAPI

### 5.3 ML 파이프라인 방향 (수요 예측)

```
데이터 수집 (release_order_items 이력)
    ↓
Feature Engineering
├── 상품별 출고 시계열
├── 계절성 (월/요일)
├── 화주사별 패턴
└── 외부 이벤트 (공휴일 등)
    ↓
모델 학습 (Prophet / LSTM / XGBoost)
    ↓
예측 결과 → WMS DB 저장
    ↓
대시보드 시각화 + 알림 트리거
```

### 6.1 투자 대비 효과 매트릭스

```
          높음

            │  1.5 KPI 대시보드   3.2 스마트 슬로팅

          낮음
            └─────────────────────────────────
              낮음              높음
                       구현 복잡도
```

### 6.2 단계별 핵심 성과 지표 (KPI)

| Phase | 핵심 KPI | 목표 |
|-------|---------|------|
| Phase 1 | 재고 정확도 | 99.5% 이상 |
| Phase 2 | 자연어 질의 응답 정확도 | 90% 이상, 이상 감지 정밀도 85% |
| Phase 3 | 피킹 생산성 향상 | 20% 이상, 보충 지시 자동화율 70% |
| Phase 4 | 완전 자동화 작업 비율 | 50% 이상 |
| Phase 5 | 플랫폼 파트너 수 | 10개사 이상 |

### 6.3 Operato WMS 차별화 포인트

글로벌 대형 WMS 벤더(Oracle, SAP)와 경쟁하는 것이 아닌, **3PL 특화 AI WMS**로 포지셔닝:

1. **3PL 특화 AI**: 화주사별 독립 AI 모델 (각 화주사 데이터로 학습)
2. **한국 시장 최적화**: 국내 택배사 API 완전 통합, 한국어 WMS Copilot
3. **빠른 온보딩**: AI 기반 신규 화주사 설정 자동화 (현재 수동)
4. **중소 물류사 접근성**: 대형 WMS 대비 합리적인 가격의 AI 기능 제공
