## 1. 개요

### 1.1 목적

유통가공(VAS) 모듈의 전체 프로세스를 효율적으로 관리할 수 있는 화면 구성을 설계합니다.

### 1.2 설계 원칙

1. **프로세스 중심 설계**: 작업 흐름에 따른 직관적인 화면 구성
2. **상태 시각화**: 작업 상태를 명확하게 표시
3. **실시간 모니터링**: 작업 진행 현황 실시간 추적
4. **터치 친화적**: PDA/태블릿 환경 최적화
5. **메타데이터 기반**: Things Factory의 MetaGrist 활용

### 1.3 대상 사용자

- **관리자**: 작업 계획, 승인, 모니터링
- **현장 작업자**: PDA 작업 수행
- **품질 관리자**: 실적 분석, 불량 관리

## 2. 화면 구성 개요

### 2.1 화면 맵

```
vas-home (대시보드)
├─ BOM 관리
│  ├─ vas-bom-list (BOM 목록)
│  └─ vas-bom-detail (BOM 상세)
├─ 작업 지시
│  ├─ vas-order-list (작업 지시 목록)
│  ├─ vas-order-new (작업 지시 생성)
│  └─ vas-order-detail (작업 지시 상세)
├─ 자재 준비
│  └─ vas-material-preparation (자재 준비 관리)
├─ 작업 진행
│  ├─ vas-work-monitor (진행 모니터링)
│  └─ vas-work-detail (작업 상세)
├─ 실적 관리
│  ├─ vas-result-list (실적 목록)
│  └─ vas-result-analysis (실적 분석)
└─ PDA 작업
   ├─ vas-pda-pick (자재 피킹)
   ├─ vas-pda-work (작업 수행)
   └─ vas-pda-result (실적 등록)
```

### 2.2 프로세스 흐름과 화면 매핑

| 프로세스 단계 | 화면 | 사용자 |
|--------------|------|--------|
| 1. BOM 등록 | vas-bom-list, vas-bom-detail | 관리자 |
| 2. 작업 지시 생성 | vas-order-new | 관리자 |
| 3. 작업 지시 승인 | vas-order-list | 관리자 |
| 4. 자재 준비 관리 | vas-material-preparation | 창고 관리자 |
| 5. 자재 피킹 | vas-pda-pick | 현장 작업자 |
| 6. 작업 수행 | vas-pda-work | 현장 작업자 |
| 7. 실적 등록 | vas-pda-result | 현장 작업자 |
| 8. 진행 모니터링 | vas-work-monitor | 관리자 |
| 9. 실적 분석 | vas-result-analysis | 품질 관리자 |

### 2.3 VAS 주문 상태 전이

VAS 주문(`VasOrder`)은 총 **7개의 상태**를 가지며, 다음과 같은 라이프사이클을 따릅니다.

#### 2.3.1 상태 정의 및 색상

| 상태 | 한글명 | 설명 | 색상 | 코드 |
|------|--------|------|------|------|
| `PLAN` | 계획 | 초기 주문 등록 상태 | 회색 (#9E9E9E) | VasOrder.java:72 |
| `APPROVED` | 승인 | 주문 승인 완료 | 파란색 (#2196F3) | VasOrder.java:72 |
| `MATERIAL_READY` | 자재준비 | 작업용 자재 준비 완료 | 녹색 (#4CAF50) | VasOrder.java:72 |
| `IN_PROGRESS` | 작업중 | VAS 작업 진행 중 | 주황색 (#FF9800) | VasOrder.java:72 |
| `COMPLETED` | 완료 | 작업 완료 | 보라색 (#9C27B0) | VasOrder.java:72 |
| `CLOSED` | 마감 | 주문 마감 (출고 완료) | 진한 회색 (#424242) | VasOrder.java:72 |
| `CANCELLED` | 취소 | 주문 취소 | 빨간색 (#F44336) | VasOrder.java:72 |

#### 2.3.2 정상 플로우

```
PLAN → APPROVED → MATERIAL_READY → IN_PROGRESS → COMPLETED → CLOSED
```

**1️⃣ PLAN (계획)**
- **진입**: 주문 최초 생성 시 자동 설정 (`VasOrder.beforeCreate()`)
- **화면**: vas-order-detail (주문 등록)
- **담당**: 관리자/영업팀
- **자동 생성**: `vasNo` (VAS-YYYYMMDD-XXXXX 형식)

**2️⃣ APPROVED (승인)**
- **전환 조건**: 관리자가 주문 승인 처리
- **화면**: 주문 상세 화면
- **담당**: 관리자
- **비즈니스 로직**: 주문 내역 검토 → 승인 버튼 클릭

**3️⃣ MATERIAL_READY (자재준비)**
- **전환 조건**: 작업에 필요한 자재(원자재, 부자재) 준비 완료
- **화면**: 자재 준비 화면 (예정)
- **담당**: 창고 관리자
- **비즈니스 로직**: BOM 기반 자재 할당 → 로케이션 지정

**4️⃣ IN_PROGRESS (작업중)**
- **전환 조건**: PDA에서 작업 시작
- **화면**: vas-work (PDA 작업 화면)
- **담당**: 작업자 (PDA)
- **비즈니스 로직**: 작업 지시서 스캔 → 작업 시작

**5️⃣ COMPLETED (완료)**
- **전환 조건**: 모든 작업 항목 완료 처리
- **화면**: PDA 작업 화면
- **담당**: 작업자 (PDA)
- **비즈니스 로직**: 작업 수량 입력 → 완료 확인

**6️⃣ CLOSED (마감)**
- **전환 조건**: 완성품 출고 처리 완료
- **화면**: 출고 관리 화면
- **담당**: 관리자/출고 담당자
- **비즈니스 로직**: 출고 지시 생성 → 피킹 완료 → 마감

#### 2.3.3 취소 플로우

```
모든 상태 → CANCELLED
```

- **전환 가능**: 어느 상태에서든 취소 가능 (단, CLOSED 이후는 권장하지 않음)
- **화면**: 주문 상세 화면
- **담당**: 관리자
- **비즈니스 로직**: 취소 사유 입력 → 확인

#### 2.3.4 칸반 보드 시각화

vas-list 화면에서 7개의 컬럼으로 칸반 스타일 표시:

```
┌─────────┬─────────┬──────────┬──────────┬─────────┬────────┬─────────┐
│  계획   │  승인   │ 자재준비 │  작업중  │  완료   │  마감  │  취소   │
│  PLAN   │APPROVED │MATERIAL_ │IN_       │COMPLETED│ CLOSED │CANCELLED│
│  회색   │  파란색 │  READY   │PROGRESS  │  보라색 │진한회색│  빨간색 │
│         │         │  녹색    │  주황색  │         │        │         │
└─────────┴─────────┴──────────┴──────────┴─────────┴────────┴─────────┘
```

#### 2.3.5 주요 특징

1. **자동 초기화**: 주문 생성 시 `status`가 null이면 자동으로 `PLAN` 설정
2. **자동 채번**: `vasNo`는 `VAS-YYYYMMDD-XXXXX` 형식으로 자동 생성
3. **권한 제어**: 각 상태 전환은 역할별로 제한 (관리자/작업자/PDA)
4. **시각적 구분**: 상태별로 고유한 색상 코드 적용
5. **추적 가능**: 모든 상태 변경 이력 추적 가능 (향후 audit 테이블 연동 예정)

상태 전이 로직은 `VasTransactionService.java`에서 구현되며, 각 화면에서 적절한 권한과 조건을 체크합니다.

#### 주요 기능

**1. BOM 기반 자재 소요 계산**
```javascript
// 자재 소요량 = BOM 단위 수량 × 주문 수량
GET /rest/vas_trx/material-preparation/requirements?vas_no={vasNo}
```

**응답**:
```json
{
  "vasNo": "VAS-20260320-001",
  "materials": [
    {
      "materialType": "MAIN",
      "skuCd": "MAT-001",
      "materialName": "상자",
      "requiredQty": 100,
      "availableQty": 250,
      "shortageQty": 0
    }
  ]
}
```

**2. 자동 할당 (FEFO 원칙)**
- First Expiry First Out: 유효기한이 가까운 순서대로 할당
- 로케이션 우선순위: VAS 작업 로케이션과 가까운 위치 우선
- LOT 단위 할당

```javascript
POST /rest/vas_trx/material-preparation/allocate
{
  "vasNo": "VAS-20260320-001",
  "strategy": "FEFO"
}
```

**3. 피킹 지시 생성**
```javascript
POST /rest/vas_trx/material-preparation/create-picks
{
  "vasNo": "VAS-20260320-001",
  "toLocCd": "VAS-WORK-01"
}
```

**응답**: `vas_material_picks` 테이블에 피킹 지시 INSERT

**4. 준비 완료 처리**
- **조건 검증**:
  1. 모든 자재 할당 완료
  2. 부족 자재 없음
- **상태 전환**: `vasOrder.status = 'MATERIAL_READY'`
- **알림**: PDA에 피킹 작업 알림 전송

```javascript
POST /rest/vas_trx/material-preparation/complete
{
  "vasNo": "VAS-20260320-001"
}
```

#### 부족 자재 처리
- 🔴 부족 수량 표시
- 대체 자재 제안 (Phase 3)
- 입고 예정 연동 (Phase 3)

#### 권한
- **창고 관리자**: 모든 기능 사용 가능
- **관리자**: 조회 및 준비 완료 처리

- ✅ 실적 등록 (`vas-pda-result`) — PDA 실적 등록, 2단계(수량입력→적치완료), OxInputBarcode 로케이션 스캔

### Phase 5: 고도화 (1주)
- ✅ 실적 분석 차트 (`vas-result-analysis`) — 유형별/일별 추이, 불량 분석, KPI
- ⬜ 바코드 스캐너 연동
- ⬜ 음성 안내
- ⬜ 오프라인 모드

### 추천 구현 순서

Phase 1 잔여 (vas-bom-list → vas-bom-detail → vas-order-list)
