## 1. 개요

### 1.1 목적

유통가공(VAS) 모듈의 전체 프로세스를 효율적으로 관리할 수 있는 화면 구성을 설계합니다.

### 1.2 설계 원칙

1. **프로세스 중심 설계**: 작업 흐름에 따른 직관적인 화면 구성
2. **상태 시각화**: 작업 상태를 명확하게 표시
3. **실시간 모니터링**: 작업 진행 현황 실시간 추적
4. **터치 친화적**: PDA/태블릿 환경 최적화
5. **메타데이터 기반**: Things Factory의 MetaGrist 활용

### 1.3 대상 사용자

- **관리자**: 작업 계획, 승인, 모니터링
- **현장 작업자**: PDA 작업 수행
- **품질 관리자**: 실적 분석, 불량 관리

## 2. 화면 구성 개요

### 2.1 화면 맵

```
vas-home (대시보드)
├─ BOM 관리
│  ├─ vas-bom-list (BOM 목록)
│  └─ vas-bom-detail (BOM 상세)
├─ 작업 지시
│  ├─ vas-order-list (작업 지시 목록)
│  ├─ vas-order-new (작업 지시 생성)
│  └─ vas-order-detail (작업 지시 상세)
├─ 작업 진행
│  ├─ vas-work-monitor (진행 모니터링)
│  └─ vas-work-detail (작업 상세)
├─ 실적 관리
│  ├─ vas-result-list (실적 목록)
│  └─ vas-result-analysis (실적 분석)
└─ PDA 작업
   ├─ vas-pda-pick (자재 피킹)
   ├─ vas-pda-work (작업 수행)
   └─ vas-pda-result (실적 등록)
```

### 2.2 프로세스 흐름과 화면 매핑

| 프로세스 단계 | 화면 | 사용자 |
|--------------|------|--------|
| 1. BOM 등록 | vas-bom-list, vas-bom-detail | 관리자 |
| 2. 작업 지시 생성 | vas-order-new | 관리자 |
| 3. 작업 지시 승인 | vas-order-list | 관리자 |
| 4. 자재 피킹 | vas-pda-pick | 현장 작업자 |
| 5. 작업 수행 | vas-pda-work | 현장 작업자 |
| 6. 실적 등록 | vas-pda-result | 현장 작업자 |
| 7. 진행 모니터링 | vas-work-monitor | 관리자 |
| 8. 실적 분석 | vas-result-analysis | 품질 관리자 |

### 3.1 VAS 대시보드 (vas-home)

#### 목적

```
┌─────────────────────────────────────────────────────┐
│  🏠 유통가공 관리 대시보드                          │
├─────────────────────────────────────────────────────┤
│                                                      │
│  📊 오늘의 작업 현황                                │
│  ┌───────────┬───────────┬───────────┬───────────┐ │
│  │  대기중   │  작업중   │   완료    │   지연    │ │
│  │    12     │     8     │    45     │     3     │ │
│  └───────────┴───────────┴───────────┴───────────┘ │
│                                                      │
│  📈 월별 실적 추이 (차트)                           │
│  ┌──────────────────────────────────────────────┐  │
│  │  [세트 구성]  [세트 해체]  [재포장]  [라벨]  │  │

│                                                      │

│  [작업 지시 생성] [작업 진행 현황] [실적 조회]     │

```

#### 주요 기능
- **실시간 현황 카드**: 상태별 작업 건수 (WebSocket 실시간 업데이트)
- **월별 실적 차트**: Chart.js 기반 라인/바 차트
- **바로가기 버튼**: 주요 화면 quick access

#### 데이터 소스
- `GET /rest/vas_orders?status=*` - 상태별 건수
- `GET /rest/vas_results?period=month` - 월별 실적

**상태 색상 코드**:
- `PLAN` (계획): Gray (#9E9E9E)

- `COMPLETED` (완료): Purple (#9C27B0)
- `CLOSED` (마감): Dark Gray (#616161)

## 4. UI/UX 컨셉

### 4.1 상태 중심 설계

#### 칸반 보드 스타일
```
[PLAN]  →  [APPROVED]  →  [MATERIAL_READY]  →  [IN_PROGRESS]  →  [COMPLETED]
  12건         8건              5건                 8건              35건
  ↓ 드래그 앤 드롭으로 상태 전환
```

#### 상태별 색상 시스템
- Material Design Color Palette 사용
- 상태 전환 애니메이션 (0.3s ease-in-out)

### 4.2 시각화 강화

#### 프로그레스 바
```javascript
<progress-bar
  value="${completedQty}"
  max="${planQty}"
  color="var(--md-sys-color-primary)"
  height="8px"
  show-label
></progress-bar>
```

#### BOM 트리 구조
```
SET-A (100 EA)
├─ SKU-A: 부품 A (200 EA)
├─ SKU-B: 부품 B (100 EA)
└─ SKU-C: 포장재 (100 EA)
```

#### 자재 소요/재고 게이지
```
SKU-A: ████████░░ 500 / 200 (충분)
SKU-B: ████░░░░░░  80 / 100 (부족)
```

### 4.3 실시간 알림

#### 알림 유형
1. **재고 부족 경고** (Warning)
   - 배경: #FFF3E0 (Orange 50)
   - 아이콘: ⚠️

2. **작업 지연 알림** (Info)
   - 배경: #E3F2FD (Blue 50)
   - 아이콘: ⏰

3. **불량률 임계치 초과** (Error)
   - 배경: #FFEBEE (Red 50)
   - 아이콘: 🚨

#### 알림 위치
- Toast (우상단, 5초 자동 사라짐)
- Badge (버튼 우상단, 개수 표시)

### 4.4 터치 친화적 (PDA/태블릿)

#### 디자인 가이드라인
- 최소 터치 영역: 44x44px
- 버튼 간격: 8px 이상
- 폰트 크기: 최소 14px
- 컨트라스트 비율: 4.5:1 이상 (WCAG AA)

#### 제스처 지원
- 스와이프: 이전/다음 작업
- 롱 프레스: 상세 정보
- 핀치 줌: 차트 확대
