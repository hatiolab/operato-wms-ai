## 1. 개요

### 1.1 목적

OMS(Order Management System) 모듈의 전체 프로세스를 효율적으로 관리할 수 있는 화면을 설계합니다.
주문 수신부터 웨이브 생성/재고 할당까지의 **계획/지시 계층**을 다루며,
실제 창고 오퍼레이션(피킹·패킹·검수·출하)은 Fulfillment 모듈 화면에서 처리합니다.

### 1.2 설계 원칙

1. **프로세스 중심 설계**: 주문 수신 → 확정 → 할당 → 웨이브 흐름에 따른 직관적 구성
2. **상태 시각화**: 11개 주문 상태를 색상으로 명확히 구분
3. **대량 처리 지원**: 일괄 확정/할당/취소로 일 수천 건 처리 효율화
4. **실시간 모니터링**: 마감 임박·재고 부족 알림으로 선제 대응
5. **메타데이터 기반**: Things Factory MetaGrist 활용

### 1.3 대상 사용자

| 사용자 | 역할 | 주요 화면 |
|--------|------|----------|
| OMS 관리자 | 주문 수신, 확정, 할당, 웨이브 생성 | 대시보드, 주문 목록/상세, 웨이브 |
| 물류 관리자 | 보충 지시 관리, 할당 모니터링 | 보충 목록, 할당 현황 |
| 데이터 운영자 | 주문 임포트, 데이터 검증 | 주문 임포트 |

## 2. 화면 구성 개요

### 2.1 화면 맵

```
oms-home (대시보드)
├─ 출하 주문 관리
│  ├─ shipment-order-list (출하 주문 목록)
│  ├─ shipment-order-detail (출하 주문 상세)
│  └─ shipment-order-import (주문 임포트)
├─ 웨이브 관리
│  ├─ shipment-wave-list (웨이브 목록)
│  └─ shipment-wave-detail (웨이브 상세)
├─ 보충 관리
│  └─ replenish-order-list (보충 지시 목록)
└─ 모니터링
   └─ stock-allocation-list (재고 할당 현황)
```

### 2.2 프로세스 흐름과 화면 매핑

| 프로세스 단계 | 화면 | 사용자 |
|--------------|------|--------|
| 1. 주문 수신 (Excel/API) | shipment-order-import | 데이터 운영자 |
| 2. 주문 확인/확정 | shipment-order-list, shipment-order-detail | OMS 관리자 |
| 3. 재고 할당 | shipment-order-list (일괄), shipment-order-detail (개별) | OMS 관리자 |
| 4. 웨이브 생성 | shipment-wave-list, shipment-wave-detail | OMS 관리자 |
| 5. 보충 지시 관리 | replenish-order-list | 물류 관리자 |
| 6. 할당 모니터링 | stock-allocation-list | 물류 관리자 |
| 7. 현황 파악 | oms-home (대시보드) | 전체 |

### 2.3 출하 주문 상태 전이 및 색상 매핑

#### 2.3.1 상태 정의 및 색상

| 상태 | 코드 | 색상 | HEX | 설명 |
|------|------|------|-----|------|
| 등록 | `REGISTERED` | 회색 | `#9E9E9E` | 주문 데이터 수신/등록 |
| 확정 | `CONFIRMED` | 파란색 | `#2196F3` | 주문 검증 완료, 할당 대기 |
| 할당완료 | `ALLOCATED` | 남색 | `#1565C0` | 재고 할당 완료, 웨이브 대기 |
| 재고부족 | `BACK_ORDER` | 빨간색 | `#F44336` | 재고 부족으로 할당 불가 |
| 웨이브 | `WAVED` | 보라색 | `#7B1FA2` | 웨이브에 포함됨 |
| 인계완료 | `RELEASED` | 인디고 | `#303F9F` | Fulfillment/WCS에 인계됨 |
| 피킹중 | `PICKING` | 주황색 | `#FF9800` | 피킹 작업 진행 중 |
| 패킹중 | `PACKING` | 연주황 | `#FFB74D` | 패킹/검수 작업 진행 중 |
| 출하완료 | `SHIPPED` | 녹색 | `#4CAF50` | 최종 출하 확정 |
| 마감 | `CLOSED` | 진한회색 | `#424242` | 정산/마감 완료 |
| 취소 | `CANCELLED` | 진빨간 | `#D32F2F` | 주문 취소 |

#### 2.3.2 상태 전이 다이어그램

```
                     OMS 주도                              이벤트 구독
                ┌──────────────────────────────────┐  ┌─────────────────────────────┐
REGISTERED → CONFIRMED → ALLOCATED → WAVED → RELEASED → PICKING → PACKING → SHIPPED → CLOSED
                 │
                 └→ BACK_ORDER (재고 부족)
                      └→ ALLOCATED (재고 확보 후 재할당)

취소 (어느 상태에서든): → CANCELLED
```

| 방식 | 용도 | 예시 |
|------|------|------|
| GraphQL | MetaGrist CRUD 조회 | `query { shipmentOrders { ... } }` |
| REST (ServiceUtil) | 트랜잭션 API 호출 | `ServiceUtil.post('/rest/oms_trx/...')` |
| REST (fetch) | 대시보드 통계 조회 | `fetch('/rest/oms_dashboard/...')` |
