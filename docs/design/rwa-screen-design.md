## 1. 개요

### 1.1 목적

반품(RWA - Return Warehouse Authorization) 모듈의 전체 프로세스를 효율적으로 관리할 수 있는 화면 구성을 설계합니다.

### 1.2 설계 원칙

1. **프로세스 중심 설계**: 반품 접수부터 처분까지 흐름에 따른 직관적 구성
2. **품질 관리 강화**: 검수 결과 및 처분 결정 명확히 표시
3. **이력 추적**: 반품 사유, 검수 결과, 처분 이력 완벽 기록
4. **실시간 모니터링**: 반품 접수 현황 실시간 추적
5. **메타데이터 기반**: Things Factory의 MetaGrist 활용

### 1.3 대상 사용자

- **고객 서비스**: 반품 요청 접수, 승인
- **입고 담당**: 반품 물품 입고 처리
- **품질 관리자**: 반품 검수, 품질 판정
- **재고 관리자**: 처분 결정 (재입고/폐기/반송)

## 2. 화면 구성 개요

### 2.1 화면 맵

```
rwa-home (대시보드)
├─ 반품 요청
│  ├─ rwa-order-list (반품 요청 목록)
│  ├─ rwa-order-new (반품 요청 생성)
│  └─ rwa-order-detail (반품 요청 상세)
├─ 반품 입고
│  ├─ rwa-receive-list (입고 대기 목록)
│  └─ rwa-receive-work (입고 작업)
├─ 반품 검수
│  ├─ rwa-inspection-list (검수 대기 목록)
│  └─ rwa-inspection-work (검수 작업)
├─ 처분 관리
│  ├─ rwa-disposition-list (처분 대기 목록)
│  └─ rwa-disposition-work (처분 결정)
├─ 실적 분석
│  ├─ rwa-result-dashboard (실적 대시보드)
│  └─ rwa-quality-analysis (품질 분석)
└─ PDA 작업
   ├─ rwa-pda-receive (반품 입고)
   └─ rwa-pda-inspect (반품 검수)
```

### 2.2 프로세스 흐름과 화면 매핑

| 프로세스 단계 | 화면 | 사용자 |
|--------------|------|--------|
| 1. 반품 요청 생성 | rwa-order-new | 고객 서비스 |
| 2. 반품 요청 승인 | rwa-order-list | 고객 서비스 |
| 3. 반품 입고 | rwa-pda-receive | 입고 담당 |
| 4. 반품 검수 | rwa-pda-inspect | 품질 관리자 |
| 5. 처분 결정 | rwa-disposition-work | 재고 관리자 |
| 6. 재고 처리 | (자동) | 시스템 |
| 7. 완료 및 마감 | rwa-order-detail | 고객 서비스 |

## 3. 화면별 상세 설계

### 3.1 RWA 대시보드 (rwa-home)

#### 목적
전체 반품 현황을 한눈에 파악

#### 레이아웃
```
┌─────────────────────────────────────────────────────┐
│  🏠 반품 관리 대시보드                              │
├─────────────────────────────────────────────────────┤
│                                                      │
│  📊 오늘의 반품 현황                                │
│  ┌───────────┬───────────┬───────────┬───────────┐ │
│  │ 요청대기  │  입고중   │  검수중   │  처분중   │ │
│  │    15     │     8     │    12     │     6     │ │
│  └───────────┴───────────┴───────────┴───────────┘ │
│                                                      │
│  📈 반품 유형별 현황 (차트)                         │
│  ┌──────────────────────────────────────────────┐  │
│  │  [고객반품]  [공급사반품]  [불량품]  [기타]  │  │
│  └──────────────────────────────────────────────┘  │
│                                                      │
│  ⚠️ 주의 항목                                       │
│  • 검수 지연: 3건 (24시간 이상)                    │
│  • 처분 대기: 6건 (48시간 이상)                    │
│                                                      │
│  🎯 바로가기                                        │
│  [반품 요청] [입고 처리] [검수 작업] [처분 결정]   │
│                                                      │
└─────────────────────────────────────────────────────┘
```

#### 주요 기능
- **실시간 현황 카드**: 상태별 반품 건수
- **유형별 통계**: 고객반품/공급사반품/불량품 등
- **알림 영역**: 지연 건수, 긴급 처리 필요 항목
- **바로가기**: 주요 작업 화면 quick access

#### 데이터 소스
- `GET /rest/rwa_orders?status=*` - 상태별 건수
- `GET /rest/rwa_orders?rwaType=*` - 유형별 통계


**상태 색상 코드**:
- `REQUEST` (요청): Gray (#9E9E9E)
- `APPROVED` (승인): Blue (#2196F3)
- `RECEIVING` (입고중): Cyan (#00BCD4)
- `INSPECTING` (검수중): Orange (#FF9800)
- `INSPECTED` (검수완료): Lime (#CDDC39)
- `DISPOSED` (처분완료): Purple (#9C27B0)
- `COMPLETED` (완료): Green (#4CAF50)
- `CLOSED` (마감): Dark Gray (#616161)
- `REJECTED` (거부): Red (#F44336)
- `CANCELLED` (취소): Red (#F44336)
